{ 下面的描述哪种最适合你当前的处境? }

  • 我更在意简单和较少的约束

    MIT许可证是一份宽松的许可证。这份许可证的文本简明扼要，只要人们承认你的所属权且不要求你承担责任。

    {% include using-sentence.html license-id="mit" %}

  • 我更在意对专利的保护

    Apache2.0许可证和MIT许可证一样是一份宽松的许可证, 但明确表达了作者的专利权和对使用者的授权。

    {% include using-sentence.html license-id="apache-2.0" %}

  • 我更在意对持续分享的保护

    GNU GPLv3是一份反版权的许可证,要求基于当前作品分发或衍生的作品都采用同样的许可证和条款。同时也明确的授予了使用者使用作品中包含专利的权利。

    {% include using-sentence.html license-id="gpl-3.0" %}
